# Library Management System - Project Statement

## Problem Statement

Traditional library operations, though increasingly digitized, require efficient and reliable software systems to manage large volumes of books and user records. The key challenge here is to design and implement a simple, console-based library management system that automates core library functions such as book management, issue and return operations, and record maintenance, while ensuring data integrity and ease of use.

## Project Scope

This project is a standalone Java console application. It is intended for single-user use, such as for librarians or individuals managing library records through a terminal. The scope is deliberately focused on core library management principles to ensure a clean implementation of object-oriented programming and multilevel inheritance in Java.

## Target Users

The main users of the system would be:

1. **Librarians:** Personnel who can add new books, manage records, and handle issue/return operations.
2. **Users (Students/Readers):** Individuals who can check book availability, issue or return books, and view book details through a self-service system.

## Features

* **Book Management:** Add and view detailed information about books in the library.
* **Issue & Return Processing:** Issue and return books with proper validation of availability.
* **Availability Inquiry:** Check the current availability status of any book at any time.
* **Record Maintenance:** Maintain and update records of all books efficiently.
* **User-Friendly Menu:** Provides an interactive and looped console menu for easy navigation.
